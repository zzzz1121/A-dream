# Dream 使用说明总览

更新时间：2026-05-28

适用版本：当前原型链路 `电脑 -> M5Stack -> ESP-NOW -> Microduino`

## 1. 当前系统链路

```text
脑电设备
  -- Bluetooth -->
电脑端 Python 桥接程序
  -- USB Serial -->
M5Stack Core ESP32
  -- ESP-NOW -->
Microduino Core ESP32
  --> DMX 灯光
  --> 步进电机
  --> 继电器 / 雾机
```

电脑端同时提供浏览器前端：

```text
浏览器前端
  -> Python 本地服务
  -> USB Serial
  -> M5Stack
  -> ESP-NOW
  -> Microduino
```

## 3. 最重要的安全逻辑

Microduino 上电后默认 `systemEnabled = false`。

在没有收到“系统开启”指令前：

- DMX 灯光保持关闭。
- 继电器 / 雾机保持关闭。
- 步进电机保持停止或禁用。
- 脑电数据不会直接驱动机器动作。

没有例外：前端手动灯光、继电器和步进电机台架调试也必须先系统开启。接入真实机械负载前，必须先确认限位、行程、方向、电流、供电和急停。

开启方式有两个：

| 来源 | 操作 |
| --- | --- |
| 浏览器前端 | 点击 `系统开启` |
| M5Stack | 按 A 键 |

关闭方式：

| 来源 | 操作 |
| --- | --- |
| 浏览器前端 | 点击 `系统关闭` 或 `全部停止` |
| M5Stack | 按 C 键关闭，按 B 键全部停止 |

## 4. 当前功能状态

| 功能 | 当前状态 |
| --- | --- |
| 脑电蓝牙 COM 读取 | 已由 Python 脚本支持 |
| ThinkGear 解析 | 已支持 `poorSignal / attention / meditation / EEG Power` |
| 浏览器实时前端 | 已支持，只显示真实状态，无真实回传时显示 `--` / 等待 |
| 前端控制命令 | 已支持，所有按钮都有发送中、已发送、失败反馈 |
| M5Stack 屏幕监测 | 已支持 |
| M5Stack 按键总开关 | 已支持 |
| M5Stack 转发 ESP-NOW | 已支持 |
| Microduino 接收 ESP-NOW | 已支持 |
| DMX 灯光 | 已启用，双 RGBW 灯地址 `001` / `005`，DMX TX 为 `GPIO5` |
| 继电器 / 雾机 | 协议和状态机已写，物理输出默认关闭 |
| 步进电机 | 单驱动板控制已支持，物理输出当前已启用于台架调试 |

继电器默认关闭物理输出，是为了避免未确认接线、触发电平和负载电源时误动作。步进电机当前已启用台架输出，正式接入真实机械负载前仍需要确认电源、驱动器电流、方向、限位、行程和急停。

当前双灯流水规则：

- 灯 1 使用当前色轮相位。
- 灯 2 相对灯 1 固定错开 `96 / 256` 圈，约 `3.36s` 相位差。
- 自动 EEG 模式下，`attention / meditation` 决定流水亮度，`attention` 会影响整体相位。
- 手动颜色模式下，前端 RGBW 会生成手动渐变流水。

前端状态显示原则：

- EEG 没有真实包时，脑电字段显示 `--`。
- M5Stack 没有真实状态行时，M5 / ESP-NOW 统计显示等待。
- Microduino 没有状态回传或掉线时，灯光、继电器、电机、安全状态显示 `--`。
- 控制按钮点击后会显示发送结果；如果 M5Stack 串口未打开，命令会失败，不会进入发送队列。

## 5. 最短运行流程

1. 给 M5Stack 上传固件。
2. 给 Microduino 上传固件。
3. 连接脑电设备到电脑蓝牙 COM。
4. 连接 M5Stack 到电脑 USB。
5. 运行电脑端脚本。
6. 打开浏览器前端。
7. 确认前端真实数据正常、M5 显示正常、MIC 在线。
8. 点击 `系统开启` 或按 M5Stack A 键。
9. 测试灯光、雾机、电机控制。
10. 结束时点击 `全部停止` 或按 M5Stack B 键。

详细步骤见 [Dream现场开机与关机流程](Dream现场开机与关机流程.md)。
