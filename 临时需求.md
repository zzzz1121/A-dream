# 临时需求

## 背景

当前作品需要同时支持有脑电参与和无脑电参与两类展演状态。现阶段现场只接了蓝牙并打开了脑电设备，因此第一步需要优先确认脑电数据是否真正进入系统，再继续完善灯光模式、配置保存和引导海报。

## 大需求 0：整体运行模式切换

目标：新增两个整体模式，用于决定作品是否依赖脑电数据。

### 模式一：有脑电

- 系统尝试读取脑电设备数据。
- 可使用脑电实时灯光。
- 可使用自动切换灯光模式。
- 前端需要展示脑电连接状态或数据状态。

### 模式二：无脑电

- 系统不依赖脑电数据运行。
- 灯光、泡泡等交互可以通过随机逻辑继续展示。
- 适合脑电设备未连接、连接不稳定或现场测试时使用。

已确认实现口径：

- 整体模式切换需要放在前端一级主界面。
- 从“有脑电”切到“无脑电”时，可以不立刻停止脑电数据监听；建议先保持监听，只在业务逻辑上忽略脑电，以便后续切回更快。
- 从“无脑电”切回“有脑电”时，需要自动重新连接或重新尝试读取脑电设备。
- “无脑电”模式不能只依赖电脑前端存在；需要考虑电脑拔掉后，板子端仍然可以继续吹泡泡和运行本地灯光逻辑。
- 电脑连接时仍需支持前端控制和联调；拔掉电脑后，作品应保留可用的本地无脑电运行能力。

## 需求 1：脑电识别通路排查
（我现在没在现场，只带了蓝牙模块和脑电）
目标：解决当前脑电识别不到的问题，重点检查蓝牙脑电设备到前端识别结果之间的数据通路。

当前现场状态：

- 已连接蓝牙。
- 已打开脑电设备。
- 系统仍然识别不到脑电。

待排查范围：

- 蓝牙设备是否真正被系统识别为可读取的数据源。
- 后端桥接程序是否正在监听正确的数据源。
- 后端是否收到原始脑电数据。
- 后端是否把脑电数据转换成前端需要的格式。
- WebSocket 或接口是否把脑电数据推送到前端。
- 前端是否正确订阅、解析并显示脑电数据。
- 脑电识别逻辑是否因为字段名、阈值、采样频率或数据为空导致无法触发。

预期结果：

- 找到脑电识别不到的具体断点。
- 修复可修复的问题。
- 在前端或后端保留必要的连接状态、最近数据时间、数据为空提示，方便现场判断是不是设备或通路问题。

## 需求 2：灯光模式与色彩逻辑

目标：灯光需要支持三种模式，并基于色板容器管理颜色范围、两灯差值和流水灯周期。

### 灯光模式一：脑电实时

- 根据脑电实时数据控制灯光。
- 使用两个主要脑电数据做二维分析，计算当前灯光颜色。
- 每 2 秒读取一次当前最新脑电数据，并触发一次色彩变化。
- 色彩变化需要柔和过渡，避免突然跳色。
- 适用于整体模式为“有脑电”，且脑电数据稳定的情况。

已确认实现口径：

- 两个主要脑电字段由实现时选择，优先选择在不同人之间或同一人不同状态下变化明显的字段。当前建议优先使用 `attention` 和 `meditation`，因为这两个字段语义清晰、现场解释成本低，也比较适合做二维映射。
- 二维分析映射到颜色时，以视觉效果好看为优先。建议接入当前色板做插值或取样，而不是固定死颜色坐标，这样色板容器可以同时服务脑电实时和随机灯光。
- 柔和变化过渡时长先使用 2 秒，与脑电读取和色彩变化节奏保持一致。

### 灯光模式二：色板随机

- 随机灯光不是单色固定逻辑。
- 每次触发泡泡时，从当前选中色板中随机选取颜色。
- 随机颜色需要遵守色板配置。
- 不依赖脑电数据，可用于无脑电模式或测试模式。

色板配置包含：

- 颜色选择。
- 多个色标或颜色节点。
- 色标位置。
- 两灯差值设置。
- 流水灯周期设置。

### 灯光模式三：自动切换

- 自动检测脑电数据是否可用。
- 检测到有效脑电数据时，使用“脑电实时”逻辑。
- 检测不到有效脑电数据时，使用“色板随机”逻辑。
- 这个模式用于现场稳定性兜底，避免脑电断开后作品停住。

已确认实现口径：

- “检测不到脑电”不是单纯指没有数据，而是指泡泡触发时检测不到正常、合理、可用于控制灯光的脑电信号。
- 是否使用脑电实时灯光，应在泡泡触发时根据当前脑电信号质量决定。
- 自动切到随机后，如果脑电恢复，不需要立即切回实时；等待下一次泡泡触发时再重新判断。
- 信号合理性建议至少结合最近数据时间和 `poorSignal` 判断，避免有数据但信号不可用时误判为有效脑电。

## 需求 3：前端灯光模式切换

目标：前端新增灯光模式切换控件，模式改为以下三项。

模式：

- 脑电实时。
- 色板随机。
- 自动切换。

前端要求：

- 当前灯光模式有明确选中状态。
- 当前整体模式也需要显示为“有脑电”或“无脑电”。
- 在无脑电整体模式下，脑电实时模式应禁用或给出不可用状态。
- 切换灯光模式后，后端灯光策略同步更新。
- 触发泡泡时，如果处于色板随机或自动切换的随机状态，需要从当前色板中随机选色。
- 灯光控制区域需要新增一个“测试脑电灯光颜色”的测试按钮，用于根据当前最新脑电数据和当前色板，单独测试脑电实时灯光映射出的颜色。
- 测试按钮只用于灯光颜色测试，不触发泡泡流程、烟雾机、风扇或步进电机。
- 如果当前没有正常、合理的脑电信号，测试按钮需要给出明确不可测试或信号不可用提示。

需要修正的理解：

- 原来的“脑电状态随机”模式不采用。
- 第三种模式改成“自动切换”：有脑电时实时，无脑电或检测不到脑电时随机。

## 需求 4：泡泡并行流程配置保存容器

目标：解决泡泡并行流程无法保存的问题，新增配置容器，用于保存不同的泡泡流程配置。

保存位置：

- 配置保存在后端服务。
- 不需要导入导出。
- 每个配置只需要支持命名即可。

预期能力：

- 新建配置。
- 保存当前配置。
- 加载已有配置。
- 重命名配置。
- 删除配置。
- 保存失败时给出明确提示。

前端结构：

- 泡泡流程配置容器作为网页二级界面。
- 可以进入配置列表，再选择某个配置进行编辑。
- 不同配置之间互不覆盖。

已确认实现口径：

- 泡泡流程配置具体包含哪些字段，需要根据现有代码结构排查后确定。
- 不需要额外创建默认配置；第一次打开为空可以接受。

## 需求 5：色板配置保存容器

目标：新增一个色板容器，用于保存不同色板，并供灯光随机和脑电实时映射使用。

保存位置：

- 色板配置保存在后端服务。
- 每个色板需要支持命名。
- 不需要导入导出。

色板内容：

- 色板名称。
- 多个颜色节点。
- 每个颜色节点包含颜色值和位置。
- 两灯差值设置。
- 流水灯周期设置。

前端结构：

- 色板容器作为网页二级界面。
- 色板创建和编辑样式参考用户提供的截图，类似渐变色标编辑器。
- 支持新建、保存、加载、重命名、删除色板。
- 灯光模式可以选择当前使用的色板。

已确认实现口径：

- 两灯差值指时间差。
- 流水灯周期单位为秒。
- 色标数量允许自由增减。

## 需求 6：A4 交互引导海报

目标：制作一个 A4 大小的交互引导海报，白底、简洁、高级，并符合当前作品风格。

设计方向：

- 白底。
- 简洁克制。
- 高级展陈感。
- 信息清晰，适合观众快速理解如何互动。
- 可以直接使用 image2 生图，也可以根据项目情况做成 HTML 或图片。

建议内容：

- 作品名称或核心概念标题。
- 观众互动步骤。
- 有脑电和无脑电两种体验方式的提示。
- 泡泡触发与灯光反馈的关系。
- 可选二维码或现场提示。

已确认实现口径：

- 海报不需要中英双语，优先使用中文。
- 作品名称：Dream。
- 可用主标题或副标题：脑电感知 | 意识转译 | 光影交互 | 近未来风格。
- 当前没有明确必须使用的字体、logo 或主视觉元素；设计时以白底、简洁、高级展陈感为主。
- 展板项目介绍可作为海报文案参考，但 A4 交互引导海报应更短、更适合观众快速阅读：

## 建议实施顺序

1. 排查脑电数据通路，确认蓝牙脑电数据是否进入后端和前端。
2. 增加整体运行模式：“有脑电”和“无脑电”。
3. 重构灯光策略为三种模式：“脑电实时”“色板随机”“自动切换”。
4. 新增色板配置容器，并把随机灯光和脑电灯光接入当前色板。
5. 新增泡泡流程配置容器，保存不同命名配置到后端。
6. 前端补齐整体模式、灯光模式、色板容器、泡泡配置容器的二级界面。
7. 制作 A4 白底交互引导海报。

## 已决策的问题

1. 脑电实时灯光使用的两个主要脑电字段由实现时选择，当前建议优先使用 `attention` 和 `meditation`。
2. “检测不到脑电”按“泡泡触发时没有正常、合理脑电信号”判断，不只按有没有数据判断。
3. 两灯差值指时间差。
4. 流水灯周期单位为秒。
5. 海报不需要中英双语，作品名称为 Dream。

## 重要实现约束：电脑可拔掉

没有脑电的模式是为了方便没有电脑在现场时，依旧可以吹泡泡玩。实现时需要注意：

- 电脑在现场时，可以通过前端进行控制、配置和联调。
- 电脑拔掉后，其他板子仍应具备基本运行能力，至少可以触发泡泡流程，并使用本地随机或流水灯光逻辑继续展示。
- “无脑电模式”需要尽量下沉到板子端或通过板子端默认逻辑兜底，不能只做成电脑前端里的状态。
- 电脑端配置和板子端运行之间需要考虑同步策略，避免拔掉电脑后配置丢失或作品不可用。
